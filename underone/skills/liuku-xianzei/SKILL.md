## 六库具象化：六腑六阶段消化管道

```
[六库消化管道 V9]

信息输入
    ↓
【胃】纳气 —— 接收原始信息，初步容量判断
  → 信息过量(>5000字) → 触发"胃胀"预警
  → 正常 → 向下传递
    ↓
【小肠】分解 —— 拆解信息结构，提取核心论点
  → 核心论点检测 (关键词匹配20+)
  → 无法分解 → 标记为"未消化原片"
    ↓
【大肠】过滤 —— 去伪存真，可信度分级
  → S级(权威源): 直接吸收
  → A级(专业源): 轻度标注后吸收
  → B级(一般源): 中度标注后吸收
  → C级(低可信): 高度存疑，仅暂存
    ↓
【胆】提纯 —— 精华浓缩，去冗余
  → 同主题合并，消除重复
  → 保留最高可信度版本
    ↓
【膀胱】暂存 —— 未完全消化知识临时存储
  → 进入暂存区的知识标记"待反刍"
  → 暂存超7天未反刍 → 自动丢弃
    ↓
【三焦】转化 —— 知识内化，融入已有体系
  → 与现有知识体系关联
  → 生成记忆锚点，便于检索
  → 输出：结构化知识单元

消化率 = (成功通过三焦的信息数 / 总输入信息数) × 100%
```

### 自适应保鲜期

根据领域动态和用户使用频率调整知识有效期：

```
[保鲜期自适应模型]
基础保鲜期:
  技术方案: 3-6月 / 行业数据: 6-12月 / 基础理论: 3年+
  法律法规: 实时 / 社区经验: 3月

领域动态调整:
  技术领域活跃度评估: 
    高活跃(每周>10篇新文章) → 保鲜期×0.6 (缩短)
    中活跃(每周3-10篇) → 保鲜期×1.0 (不变)
    低活跃(每周<3篇) → 保鲜期×1.5 (延长)

用户使用频率调整:
  高频使用(每月>5次查询) → 保鲜期×1.3 (延长，因为重要)
  中频使用(每月1-5次) → 保鲜期×1.0 (不变)
  低频使用(每月<1次) → 保鲜期×0.7 (缩短，因为不重要)

验证驱动调整:
  知识被验证为正确 → 保鲜期+20%
  知识被验证为过时 → 保鲜期立即归零，标记废弃
  知识未被使用超过2倍保鲜期 → 自动降级到"冷库存档"
```

### 知识图谱自动补全

基于关联推断自动补全缺失节点和边：

```
[图谱补全机制]
推断触发:
  节点A与节点B有强关联，节点B与节点C有强关联
  但节点A与节点C之间无边 → 推断A-C可能存在间接关联

推断置信度:
  强(A-B) + 强(B-C) → A-C推断置信度: 中(0.5)
  强(A-B) + 中(B-C) → A-C推断置信度: 弱(0.3)
  仅当两条边均为A级时，推断置信度可达中等级

推断标注:
  所有推断补全的边标注为"推断"(非"观测")
  推断边不用于核心洞察，仅作为"待验证线索"
  当观测数据证实推断 → 升级为正式边

孤儿节点处理:
  节点入库30天后仍无任何关联边 → 标记"孤立知识"
  孤立知识自动进入"待关联队列"，等待后续信息补充
  孤立超过2倍保鲜期 → 降级到"背景知识库"
```

### 来源可信度动态调整

根据历史准确率动态调整来源评级：

```
[来源评级进化]
初始评级: 基于来源类型(S/A/B/C)

准确率追踪:
  每次使用该来源的知识后，记录验证结果(正确/过时/错误)
  准确率 = 正确次数 / 总使用次数

动态调整:
  B级来源准确率>90%超过10次 → 升级为A级
  A级来源准确率<70%超过5次 → 降级为B级
  S级来源准确率<80% → 触发复核，可能降级
  C级来源准确率>85%超过15次 → 升级为B级

黑名单机制:
  准确率<50% → 标记"不可靠来源"
  不可靠来源权重降至0.2，使用时强制附加警告
  连续3次错误 → 暂停使用该来源，等待人工复核
```
