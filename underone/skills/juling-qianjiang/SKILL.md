# 拘灵遣将

> 统御百器，调度千灵。观灵点将，号令其行。——风天养

## 原著概念映射

| 原著概念 | 工程映射 | 说明 |
|---------|----------|------|
| 号令百鬼 | 多工具并行调度 | 同时协调多个工具/API/脚本 |
| 观灵 | 工具健康检查 | 扫描可用性、响应时间、错误率 |

```
改进动作: 降级策略优化 / 工具健康修复

[对抗性自检]
故障1: 灵体全部不可用 -> 启用降级保护或服灵内化
```

## 核心工作流

### 1. 观灵 (工具侦察)

```
[观灵]
灵体ID: __ 能力: __ 健康度: __ 可用性: __
□ 功能覆盖度: __% / 目标: 100%
□ 外部API: __项 / 自定义脚本: __项
```

### 2. 点将 (任务匹配)

```
[点将决策 V9]
任务: {子任务}
候选灵体: {ID} | 匹配度:{高/中/低} | 状态:{可用/不可用}

可用灵体 -> 正常调度
不可用灵体 -> 执行降级保护:
  1. 尝试缓存数据返回
  2. 尝试模拟响应
  3. 尝试跳过该步骤
  4. (可选) 启用服灵模式 → 生成内联替代实现
```

### 3. 降级保护 (默认策略)

```
[降级保护 V9]
触发条件: 灵体不可用(超时/失败/不存在)

保护等级:
  Level 1: 缓存返回 — 返回上次成功结果
  Level 2: 模拟响应 — 根据接口定义生成模拟数据
  Level 3: 跳过步骤 — 标记为降级，继续后续流程
  Level 4: 服灵内化 — 生成替代实现(需显式开启)

配置方式:
  strategy=protect (默认) — 使用Level 1-3
  strategy=possess (可选) — 启用Level 4服灵内化
```

### 4. 百鬼夜行 (大规模并行)

```
[百鬼夜行]
自检:
  □ 总任务量: __项
  □ 分片: __片 × __项/片
  □ 并行度: __片
  □ SLA: __秒
```

### 5. 统御监控


## 实战速查卡

| 触发条件 | 自动动作 | 目标产出 | 配置 |
|---------|---------|---------|------|
| 任务需2+工具 | 观灵点将 | 调度决策表 | - |
| 灵体不可用 | 降级保护 | 缓存/模拟/跳过 | strategy=protect(默认) |
| 灵体不可用(服灵模式) | 内化生成 | 替代实现 | strategy=possess(可选) |
| 批量处理>10项 | 百鬼夜行 | 并行结果 | - |
| SLA风险>50% | 熔断保护 | 拒绝新任务 | - |

---

## 统御原则

- **优先保护**: 默认使用降级保护，不强制内化
- **可选服灵**: 通过 strategy=possess 显式开启服灵模式
- **健康监控**: 持续追踪工具可用性和响应质量
- **熔断机制**: SLA达成率过低时主动拒绝新任务
- **自动适配**: 数据格式不匹配时自动转换

> **世界观彩蛋**: 工具不可用时建议优先降级保护，而非强制内化（服灵侵蚀）。可通过配置切换降级策略。
