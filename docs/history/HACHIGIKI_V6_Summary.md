## 一、修身炉 (XiuShenLu) 架构设计

### 灵感来源
修身炉源自《一人之下》马仙洪创造的神器——可将普通人转化为异人，核心是自主运行的"核心"球体。映射到Agent系统，修身炉是一个**自进化引擎**，让skills具备自主进化能力。

### 五大核心模块

```
XiuShenLu 修身炉
├── QiSource (炁源)
│   └── 运行时指标自动收集器
│   └── 持久化存储每skill的历史执行数据
│
├── Refiner (炼化器)
│   └── 性能瓶颈分析引擎
│   └── 健康分计算 (0-100)
│   └── 退化趋势检测
│
├── Transformer (转化器)
│   └── 三级进化执行器
│   ├── tuning (低风险): 参数自动调优
│   ├── extension (中风险): 逻辑扩展
│   └── refactor (高风险): 架构重构
│
├── Core (核心)
│   └── 进化决策引擎
│   └── 触发条件判断
│   └── 与八卦阵协调
│
└── Rollback (回退)
    └── 版本快照管理
    └── 一键安全回滚
    └── 保留最近5版本
```

### 进化触发条件

| 条件 | 阈值 | 进化类型 |
|------|------|----------|
| 成功率连续下降 | < 80% (V6灵敏度) | tuning/extension |
| 人工干预频率升高 | > 0.1/任务 | extension |
| 执行时间超标 | > 115% SLA | tuning |
| 连续质量下降 | 4次递减 | tuning |
| 用户主动请求 | N/A | 按指令执行 |

### 进化安全机制

1. **自动备份**: 进化前自动快照当前版本
2. **语法验证**: 进化后自动检查脚本语法
3. **结构验证**: 确保SKILL.md和scripts/目录完整
4. **失败回滚**: 验证失败自动回退到上一个稳定版本
5. **八卦阵协调**: 进化期间skill进入只读保护状态

## 四、自进化闭环演示

### 完整进化周期 (7步)

```
Step 1: COLLECT    → 修身炉从炁源读取skill的100条历史运行记录
Step 2: ANALYZE    → 炼化器计算健康分、检测退化趋势
Step 3: DECIDE     → 核心判断：是否需要进化？什么类型？
Step 4: EVOLVE     → 转化器执行：备份→修改→版本号更新
Step 5: VALIDATE   → 检查脚本语法+文件结构完整性
Step 6: DEPLOY     → 验证通过→部署新版本
Step 7: MONITOR    → 标记监控状态，后续10次运行持续跟踪
```
